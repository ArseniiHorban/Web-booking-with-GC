Booking System Demo

This project is a web application for booking. It demonstrates a robust, scalable architecture with its own and Google Calendar databases with one-way synchronization, background task processing, and a clean, responsive UI.

Key Features:

  • Django backend with modular project structure
  • Booking form with real-time slot validation and user-friendly interface
  • Automatic event creation in the administratorâ€™s Google Calendar
  • Events can be created, edited or deleted in Google Calendar, all changes are synched with the local database
  • Asynchronous background processing using Celery and Redis (calendar's sync)
  • All bookings in Google Calendar have specific colour, so that the administrator can create bookings not related to the program in the same calendar
  • Ready for deployment and further extension

How it works:

  1. The user selects a date and time for a booking and submits the booking form.
  2. The system checks slot availability and creates a booking record in the database and Google Calendar.
  3. All bookings are visible to the admin in both the web interface and Google Calendar.

Or, the other way:

  1. The admin creates a booking in Google Calendar setting the event color to yellow
  2. Once in 5 minutes system checks for any changes in Google Calendar and updates the local database accordingly.
  3. All bookings are visible to the admin in both the web interface and Google Calendar.

Tech Stack:

Django PostgreSQL TailwindCSS Celery Redis Google Calendar API

This demo was made as a pet project to showcase my skills in Django and web development, and further understand the integration of listed above technologies. It is not intended for production use at the moment, but can be a good starting point for building a more complex booking system and integrating it into a production-ready project.
